## aggregate_benchmark.py 第二轮修复

Agent 在 Linux 上实测暴露两个新问题：

### 问题 1：目录结构被 Agent 搞乱后脚本不兼容
Agent 删了 `with_skill/` `without_skill/` 子目录，把 `grading.json` 直接放在 `eval-N/` 根下，但脚本只扫描子目录（`with_skill/outputs/` 或 `run-*/`），不认根目录的 `grading.json`。
- **修复**：新增 Layout D 支持——当 eval-N/ 下没有任何 config 子目录时，直接读 `eval-N/grading.json`，归入 `default` 配置。

### 问题 2：grading.json 格式不兼容
Agent 手写的 grading.json 是扁平格式（`{"passed": true, "eval_name": ...}`），没有 `summary` 包装层，导致所有字段读出来都是 0。
- **修复**：`_load_grading()` 新增 flat format 检测——如果没有 `summary` 字段，从顶层 `passed`(bool) 推导出 passed/failed/pass_rate/total；如果没有 `expectations` 数组但有 `text` 字段，自动合成单个 expectation。

## skill-grader.md 启动报错修复

报错：`Configuration is invalid ... expected record, received array tools`

根因：Linux 上的 skill-grader.md 文件可能版本不一致，或 OpenCode 版本对 `tools` 字段的解析有差异。

修复：将 `tools: { write: false, edit: false, bash: false }` 改为 `permission: { write: deny, edit: deny, bash: deny }`。`permission` 是更推荐的细粒度控制方式，语义也更明确。

### 根因追加发现

报错持续的原因是 `.opencode/agents/` 下有旧版副本，用的是**数组格式** `tools: [read_file, search_content]` 和 `role: subagent`（非标准字段）。OpenCode 加载项目级 `.opencode/agents/` 优先于全局 `~/.config/opencode/agents/`，所以之前改全局的没用。

最终修复：4 个 agent 文件全部统一，去掉 `tools`/`role` 字段，只用 `mode` + `permission`。

| 文件 | frontmatter |
|------|-------------|
| `~/.config/opencode/agents/skill-executor.md` | `mode: primary`（去掉 tools） |
| `~/.config/opencode/agents/skill-grader.md` | `mode: subagent` + `permission: deny` |
| `skill-creator/.opencode/agents/skill-executor.md` | `mode: primary`（去掉旧版 tools 数组） |
| `skill-creator/.opencode/agents/skill-grader.md` | `mode: subagent`（去掉旧版 tools 数组） |
